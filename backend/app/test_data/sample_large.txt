Vagmi Retrieval System Documentation

The retrieval system is responsible for processing uploaded documents and making them searchable. The system begins by extracting text from supported file formats including TXT, DOCX, and PDF documents. Once the text has been extracted, it is cleaned and normalized to remove excessive whitespace and formatting inconsistencies.

The chunking service then divides the document into smaller overlapping segments. Chunking is important because embedding models have context limits and because smaller pieces of information generally produce more relevant retrieval results. Each chunk overlaps with the previous chunk to preserve context across chunk boundaries.

The embedding service converts chunks into vector representations using a sentence transformer model. These vectors capture semantic meaning and enable similarity search. ChromaDB stores both the vectors and associated metadata such as document identifiers and ownership information.

When a user submits a query, the query is embedded using the same embedding model. ChromaDB performs a similarity search against stored vectors and returns the most relevant chunks. These chunks can then be used as context for downstream language model processing.

Document isolation is an important requirement of the system. Users should only be able to retrieve information from documents they own or are authorized to access. Metadata filtering ensures that search results respect ownership boundaries.

The retrieval pipeline should be modular, testable, and easy to debug. Each component should have a single responsibility and a clearly defined interface. This architecture simplifies maintenance and future enhancements.
